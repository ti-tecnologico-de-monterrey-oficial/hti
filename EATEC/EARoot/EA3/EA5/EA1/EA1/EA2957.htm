api_portalws leeCookieUsuarioAutenticado
Details:
Type: Sequence Synchronous Call
Busca la cookie que identifica al usuario autenticado en el portal, en el siguiente orden (Solo si no existe esa cookie se busca la siguiente): ¿Existe IdUserPortal? SI: UsuarioAutenticado = cookie IdUserPortal NO: UsuarioAutenticado = cookie MailUserPortalAlumno ---------------------- 1. IdUserPortal Contiene el valor encriptado del id cuando el usuario es un padre de familia 2. MailUserPortalAlumno Contiene el valor encriptado de la matrícula cuando el usuario es un alumno. -------------------------------------------- Nota: para el caso de colaboradores su cookie (MailUserPortal) se lee justo antes de hacer el llamado al ws.
api_portalws calculaServidorPortal
Details:
Type: Sequence Synchronous Call
Para personalizar la apariencia del servicio de acuerdo al usuario autenticado se cargan diferentes tipos de letra, imágenes, íconos, estilos, etc., este paso identifica el tipo de usuario de acuerdo a la cookie SessionPortal y entrega el servidor web que corresponde
