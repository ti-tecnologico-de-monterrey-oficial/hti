El sistema debe de realizar la asignación del Estatus de Beca a VENCIDA El rango de aceptación actual es: Días de comunicación + 7 días naturales <br/>(Tiempo configurable por administrador)<br/>El sistema debe de Mostrar Ayuda: Terminó el tiempo para aceptar la beca El candidato ya no tendrá opción para aceptar la beca que está VENCIDA  y el sistma debe de verificar que tenga el pago de anticipo de colegiatura El pago de anticipo de colegiatura se toma de la configuración existente en 9.3.1 (B10.5.1)<br/><br/><ol>
<li>El candidato no cuenta con pago de anticipo de colegiatura</li></ol>
En caso de que no tenga el pago de anticipo de colegiatura el sistema debera de Mostrar mensaje configurable para indicarle al candidato que es necesario cubrir la cuota de anticipio de colegiatura para poder solicitar extensión de tiempo de beca <br/><br/>2. El candidato cuenta con pago de anticipo de colegiatura<br/>En caso de que el candidato si cuente con el pago de anticipo de colegiatura el sistema debera de mostrarle al candidato la opción para Solicitar extensión de tiempo a Becas del Campus <br/><br/>El sistema debe de permitir al responsable de becas del campus que pueda Seleccionar Autorizar y agregar comentarios Agregar "n" días naturales para hacer los cambios por alumno<br/>e) El sistema debera de Habilitar automáticamente  la opción de aceptar el Apoyo  para que el candidato pueda aceptar o activar su apoyo educativo<br/><br/>
